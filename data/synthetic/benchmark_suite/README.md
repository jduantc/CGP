# Benchmark suite datasets

Generated by `scripts/run_benchmark_suite.py` via `cgp/datagen.py`, one
folder per (true graph pair, mechanism, seed) combination it tests. Each
folder contains:

- `merged.dat` -- the mixture dataset (both components concatenated)
- `true_subset1.dat`, `true_subset2.dat` -- the ground-truth split
- `meta.json` -- the two true DAGs (as edge lists and index into
  `cgp.probing.THREE_NODE_EQUIVALENCE_CLASSES`), structural distance
  between them, mechanism, seed, and points-per-component

Folder names follow `pair{i:02d}-{j:02d}_{mechanism}_seed{seed}`, matching
the `dataset` column in `results/benchmark_suite.csv` so a CSV row's
`cgp_rank`/`em_rank` can be traced back to the exact data that produced it.

Regenerating `scripts/run_benchmark_suite.py` reproduces these files
byte-for-byte (the RNG seeding is deterministic), but they're committed
directly so they serve as a fixed, inspectable test set independent of any
future change to `cgp.datagen`'s internal random-number usage.
